2. MySQL 이미지 설치

잘 설치 되었는지 docker에 다운된 이미지 확인하기

이미지가 다운 받아졌다면 도커의 대시보드 이미지 탭에서도 확인 가능하다.

3. Mysql 컨테이너 생성

  • –name : 컨테이너 이름을 생성, 실행한다.
  • e : 컨테이너 내에서 사용할 환경변수를 설정한다.
  • MYSQL_ROOT_PASSWORD= : MYSQL의 ROOT 권한의 비밀번호를 설정한다.
  • -d : detach 모드로 컨테이너를 실행한다. 컨테이너가 백그라운드로 실행되도록 한다.
  • -p <호스트 포트> <컨테이너 포트> : 호스트와 컨테이너의 포트를 연결한다.
  • mysql:latest : 컨테이너에 사용할 이미지 -> 도커 대시보드 이미지 탭에서 확인해보면 tag가 latest로 되어있다.

잘 생성되었는지 컨테이너 목록을 조회해 볼 수 있다.

잘 생성 되었다면 대시보드에서도 컨테이너를 조회할 수 있다.

4. 컨테이너의 MySQL 접속하기

컨테이너 대시보드에서도 컨테이너와 연결된 터미널을 열 수 있다.

5. Workbench에서 docker에 띄운 MySQL 연결하기

MySQL Connections 옆의 + 아이콘을 클릭하여 New Connection을 작성한다.
docker mysql image에서 설정해준 포트 번호를 써주고 username과 password도 설정해준다.
Test Connection을 실행하여 연결에 문제가 없는지 확인한다.

같은 경고 메시지가 떴지만.. 일단은 프로젝트에 연결하여 서버 실행하는 것이 목적이었기 때문에 무시하고 넘겼다. 후에 진행 중 문제가 생긴다면 다시 해결해야겠다.

workbench를 통해 DB를 생성했다면, 인텔리제이의 데이터베이스 탭을 사용하여 생성한 DB를 연결해준다.
연결 테스트 성공한 후 적용 -> 확인 해주면 연결 완.

이 후 run 하면 에러 없이 서버가 돌아간다.
